# Extended User Guide

This extended user guide provides in-depth explanations of the architecture, design patterns, and workflows used in the `noads` package.

## Overview

The `noads` package is designed to support the optimization of aviation decarbonization scenarios. The repository demonstrates how object-oriented programming principles enable modular and dynamic workflows for complex multidisciplinary optimization problems.

## Contents

- [Object-Oriented Architecture](oop_architecture.md) - Demonstrates the benefits of OOP for managing complex models
- [Workflow Management](workflow_management.md) - How to build modular and dynamic optimization workflows
- [Core Concepts](core_concepts.md) - Fundamental concepts and design patterns

## Key Features

- **Modular Design**: Models are composed of independent, reusable components
- **Dynamic Workflows**: Workflows can be assembled and modified programmatically
- **Type Safety**: Strong typing with Python type hints
- **Automatic Differentiation**: Integration with JAX for efficient gradient computation
- **Multidisciplinary Optimization**: Leverages GEMSEO for handling coupled systems
